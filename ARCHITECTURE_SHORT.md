# Архитектура системы управления складом

## Трехслойная архитектура

```
┌──────────────────────────────────────────────────┐
│  СЛОЙ ПРЕДСТАВЛЕНИЯ (Presentation)               │
│  • HTML-страницы (Jinja2 шаблоны)                │
│  • CSS стили + JavaScript                        │
│  • Отображение данных пользователю               │
├──────────────────────────────────────────────────┤
│  СЛОЙ БИЗНЕС-ЛОГИКИ (Business Logic)             │
│  • REST API (FastAPI)                            │
│  • Проверка прав доступа (роли и разрешения)     │
│  • Валидация входных данных (Pydantic)           │
│  • Обработка запросов                            │
├──────────────────────────────────────────────────┤
│  СЛОЙ ДАННЫХ (Data)                              │
│  • База данных PostgreSQL (17 таблиц)            │
│  • ORM модели (SQLAlchemy)                       │
│  • Автоматические триггеры и аудит               │
└──────────────────────────────────────────────────┘
```

## Безопасность

- Хеширование паролей + сессии
- Контроль доступа: Сотрудник → Роль → Разрешения
- Логирование всех действий

## Стек

FastAPI • PostgreSQL • SQLAlchemy • Pydantic • Jinja2
